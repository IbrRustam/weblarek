# Интернет-магазин «Web-Larёk»

«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component

Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`

#### Класс Api

Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter

Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` - хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

### Данные и модели данных (Слой данных / Model)

В этом слое описаны структуры данных, которые приходят с сервера и используются внутри приложения, а также классы-модели, отвечающие за их хранение, изменение и валидацию.

#### Интерфейсы и типы данных

- `TPayment` — тип данных, описывающий доступные способы оплаты: может принимать значения `'card'` или `'cash'`.
- `IProduct` — интерфейс, описывающий структуру карточки товара (id, описание, ссылка на картинку, название, категория, цена). Цена может быть `null`, если товар недоступен.
- `IBuyer` — интерфейс данных покупателя, собираемых при оформлении заказа (способ оплаты, email, телефон, адрес доставки).
- `IProductList` — тип объекта, возвращаемого сервером при запросе списка товаров (массив товаров `items` и их общее количество `total`).
- `IOrder` — интерфейс полного заказа, отправляемого на сервер. Расширяет `IBuyer`, добавляя массив ID купленных товаров `items` и итоговую сумму `total`.
- `IOrderResult` — объект ответа сервера при успешной оплате (содержит ID созданного заказа и списанную сумму).

#### Класс ProductsData

Класс отвечает за хранение и работу с каталогом товаров, полученных с сервера, а также за управление товаром, выбранным для подробного просмотра.
_Реализует интерфейс `IProductsData`._

- **Поля:**
  - `protected _products: IProduct[]` — массив объектов всех товаров.
  - `protected _preview: IProduct | null` — товар, выбранный пользователем для отображения в модальном окне.
- **Методы и свойства:**
  - `setProducts(products: IProduct[]): void` — сохраняет массив товаров в модель.
  - `getProduct(productId: string): IProduct | undefined` — возвращает товар по его уникальному ID.
  - геттер `products: IProduct[]` — возвращает весь список товаров.
  - геттер/сеттер `preview: IProduct | null` — свойство для установки и получения товара, выбранного для превью (реализовано в едином стиле свойств класса).

#### Класс BasketData

Класс инкапсулирует логику работы с корзиной покупок. Отвечает за добавление, удаление товаров и подсчет стоимости.
_Реализует интерфейс `IBasketData`._

- **Поля:**
  - `protected _items: IProduct[]` — массив товаров, добавленных в корзину.
- **Методы:**
  - `add(item: IProduct): void` — добавляет товар в корзину, если его там еще нет.
  - `remove(id: string): void` — удаляет товар из корзины по его ID.
  - `clear(): void` — полностью очищает корзину (удаляет все товары).
  - `has(id: string): boolean` — проверяет, находится ли товар с данным ID в корзине.
  - геттер `items: IProduct[]` — возвращает список товаров в корзине.
  - геттер `total: number` — возвращает суммарную стоимость всех товаров в корзине.
  - геттер `count: number` — возвращает количество товаров в корзине.

#### Класс UserData

Класс отвечает за хранение персональных данных покупателя. Класс спроектирован как чистая модель данных и не хранит в себе состояние ошибок интерфейсных форм.
_Реализует интерфейс `IUserData`._

- **Поля:**
  - `protected _payment: TPayment | null` — выбранный способ оплаты (`card` или `cash`), на старте равен `null`.
  - `protected _address: string` — адрес доставки.
  - `protected _email: string` — электронная почта покупателя.
  - `protected _phone: string` — номер телефона покупателя.
- **Методы:**
  - `setField(field: keyof IBuyer, value: string): void` — сохраняет значение в указанное поле данных пользователя (выполняет только функцию записи).
  - `getUserData(): IBuyer` — возвращает чистый объект с данными покупателя для отправки на сервер.
  - `clear(): void` — очищает все данные пользователя.
  - `validate(): BuyerErrors` — выполняет валидацию данных покупателя. Метод вычисляет ошибки «на лету» (динамически) и сразу возвращает объект с найденными ошибками наружу, соблюдая принцип единственного источника истины.
  - Геттеры для полей `payment`, `address`, `email`, `phone` для безопасного чтения данных из внешнего кода.

### Слой коммуникации

Этот слой отвечает за непосредственное сетевое взаимодействие с сервером API, изолируя логику запросов от моделей данных.

#### Класс LarekApi

Служит прослойкой между сервером интернет-магазина и внутренними моделями приложения. Класс спроектирован с соблюдением принципа инверсии зависимостей (Dependency Inversion) и зависит от абстрактного интерфейса, а не конкретного класса сетевого клиента.
_Реализует интерфейс `ILarekApi`._

- **Поля:**
  - `protected _api: IApi` — экземпляр базового класса для отправки HTTP-запросов (работает через интерфейс-абстракцию `IApi`).
- **Конструктор:**
  - `constructor(api: IApi)` — принимает в качестве зависимости любой объект, реализующий интерфейс `IApi`.
- **Методы:**
  - `getProductList(): Promise<IProductList>` — запрашивает с сервера массив доступных товаров.
  - `createOrder(order: IOrder): Promise<IOrderResult>` — отправляет оформленный заказ на сервер и возвращает результат успешной транзакции.

### Слой представления (Слой отображения / View)

Этот слой отвечает за отображение интерфейса, взаимодействие с пользователем и отрисовку данных, переданных из слоя бизнес-логики. Все компоненты представления наследуются от базового класса `Component` и работают с данными через сеттеры, изолируя DOM-структуру от остальных частей приложения. Взаимодействие с Презентером происходит через генерацию событий в брокер `EventEmitter`.

#### Класс Page

Управляет глобальными элементами интерфейса на главной странице приложения: галереей товаров, корзиной и блокировкой прокрутки.
_Наследуется от базового класса `Component`._

Конструктор:  
`constructor(container: HTMLElement, events: IEvents)` — инициализирует элементы каталога, корзины, счетчика и главного контейнера страницы. Навешивает слушатель клика на кнопку корзины.

Поля класса:  
`protected _counter: HTMLElement` — элемент счетчика товаров в корзине.  
`protected _catalog: HTMLElement` — контейнер-сетка для отображения карточек товаров.  
`protected _wrapper: HTMLElement` — главный внутренний контейнер страницы.  
`protected _basket: HTMLElement` — кнопка открытия корзины.

Методы и свойства:  
сеттер `counter(value: number)` — обновляет числовое значение на иконке корзины.  
сеттер `catalog(items: HTMLElement[])` — полностью очищает старое содержимое галереи и вставляет новые DOM-элементы карточек товаров.  
сеттер `locked(value: boolean)` — включает или выключает CSS-класс блокировки скролла `.page__wrapper_locked` на обертке страницы при открытии модального окна.

#### Класс Modal

Реализует универсальный контейнер для модальных окон, управляя их открытием, закрытием и отображаемым контентом.
_Наследуется от базового класса `Component`._

Конструктор:  
`constructor(container: HTMLElement, events: IEvents)` — находит кнопку закрытия и контейнер для содержимого. Устанавливает слушатели на клик по оверлею и крестику для закрытия окна.

Поля класса:  
`protected _closeButton: HTMLButtonElement` — кнопка-крестик закрытия модального окна.  
`protected _content: HTMLElement` — контейнер для динамического содержимого модального окна.

Методы и свойства:  
сеттер `content(value: HTMLElement)` — заменяет текущее содержимое окна новым DOM-элементом.  
`open(): void` — отображает модальное окно на странице и генерирует событие `modal:open`.  
`close(): void` — скрывает модальное окно, очищает его контент и генерирует событие `modal:close`.  
`render(data: IModalData): HTMLElement` — переопределяет метод базового класса: принимает контент, вставляет его в окно и автоматически вызывает метод `open()`.

#### Класс Card (Базовый)

Абстрактный класс, содержащий общую разметку и логику управления базовыми текстовыми элементами товара (название и цена).
_Наследуется от базового класса `Component`._

Конструктор:  
`constructor(container: HTMLElement, actions?: ICardActions)` — находит элементы заголовка и цены. Если передан коллбэк клика, привязывает его к корневому элементу или целевой кнопке.

Поля класса:  
`protected _title: HTMLElement` — элемент названия товара.  
`protected _price: HTMLElement` — элемент стоимости товара.

Методы и свойства:  
сеттер/геттер `title(value: string)` — управляет текстовым содержимым названия.  
сеттер/геттер `price(value: number | null)` — выводит цену товара в формате `X синапсов`. Если цена равна `null`, подставляет текст `Бесценно`.

#### Класс CatalogCard

Отвечает за отображение сокращенной версии карточки товара в главной галерее приложения.
_Наследуется от базового класса `Card`._

Конструктор:  
`constructor(container: HTMLElement, actions?: ICardActions)` — вызывает родительский конструктор, находит элементы категории и картинки. Навешивает клик на всю область карточки.

Поля класса:  
`protected _category: HTMLElement` — элемент текстовой категории товара.  
`protected _image: HTMLImageElement` — элемент изображения товара.

Методы и свойства:  
сеттер `category(value: string)` — устанавливает текст категории и навешивает специфический CSS-класс цвета плашки на основе словаря категорий.  
сеттер `image(value: string)` — устанавливает путь к картинке через метод `setImage`.

#### Класс PreviewCard

Отвечает за развернутое отображение товара в модальном окне с детальным описанием и кнопкой покупки.
_Наследуется от базового класса `Card`._

Конструктор:  
`constructor(container: HTMLElement, actions?: ICardActions)` — находит элементы категории, картинки, описания и кнопку управления состоянием корзины.

Поля класса:  
`protected _category: HTMLElement` — элемент категории.  
`protected _image: HTMLImageElement` — элемент изображения.  
`protected _description: HTMLElement` — элемент детального текстового описания товара.  
`protected _button: HTMLButtonElement` — кнопка добавления/удаления товара из корзины.  
`protected _isOrdered: boolean` — внутренний флаг нахождения товара в корзине.

Методы и свойства:  
сеттер `description(value: string)` — заполняет элемент описания текстом.  
сеттер `isOrdered(value: boolean)` — изменяет внутренний статус и вызывает метод `updateButtonState()`.  
`protected updateButtonState(): void` — меняет текст кнопки (`В корзину` / `Удалить из корзины`). Если цена товара `null`, кнопка программно блокируется с текстом `Недоступно`.

#### Класс BasketCard

Отвечает за отображение товара в виде лаконичной строки внутри таблицы корзины.
_Наследуется от базового класса `Card`._

Конструктор:  
`constructor(container: HTMLElement, actions?: ICardActions)` — находит элемент порядкового номера и кнопку удаления. Привязывает клик удаления к кнопке.

Поля класса:  
`protected _index: HTMLElement` — элемент сквозного индекса строки.  
`protected _button: HTMLButtonElement` — кнопка удаления товара (иконка корзины).

Методы и свойства:  
сеттер `index(value: number)` — устанавливает порядковый номер товара в списке корзины.

#### Класс Basket

Управляет интерфейсом корзины, отображая список выбранных товаров, итоговую стоимость и кнопку перехода к оформлению.
_Наследуется от базового класса `Component`._

Конструктор:  
`constructor(container: HTMLElement, events: IEvents)` — находит список элементов, элемент общей стоимости и кнопку оформления. Навешивает на кнопку генерацию события `order:open`.

Поля класса:  
`protected _list: HTMLElement` — контейнер для списка строк товаров.  
`protected _total: HTMLElement` — элемент отображения общей суммы заказа.  
`protected _button: HTMLButtonElement` — кнопка оформления заказа.

Методы и свойства:  
сеттер `items(items: HTMLElement[])` — обновляет список DOM-элементов товаров в корзине. Если корзина пуста, вставляет текст `Корзина пуста` и блокирует кнопку оформления.  
сеттер `total(value: number)` — обновляет итоговую стоимость товаров.

#### Класс Form (Базовый)

Абстрактный класс, автоматизирующий обработку ввода, валидацию и вывод ошибок для всех интерактивных форм.
_Наследуется от базового класса `Component`._

Конструктор:  
`constructor(container: HTMLFormElement, events: IEvents)` — находит блок ошибок и кнопку сабмита. Навешивает реактивное отслеживание события `input` для полей и блокирует стандартную перезагрузку страницы при отправке `submit`.

Поля класса:  
`protected _submit: HTMLButtonElement` — кнопка отправки формы.  
`protected _errors: HTMLElement` — контейнер для отображения текстов ошибок валидации.

Методы и свойства:  
сеттер `valid(value: boolean)` — управляет доступностью кнопки отправки формы.  
сеттер `errors(value: string[])` — принимает массив ошибок и выводит их в текстовом формате через запятую.  
`clear(): void` — сбрасывает значения всех полей формы.

#### Класс OrderForm

Управляет первой формой оформления заказа (выбор типа оплаты и ввод адреса доставки).
_Наследуется от базового класса `Form`._

Конструктор:  
`constructor(container: HTMLFormElement, events: IEvents)` — собирает альтернативные кнопки выбора оплаты и вешает на них обработчики кликов.

Поля класса:  
`protected _buttons: HTMLButtonElement[]` — массив кнопок выбора способа оплаты (`Онлайн` / `При получении`).

Методы и свойства:  
сеттер `payment(value: string)` — подсвечивает активную кнопку оплаты CSS-классом `.button_alt-active` и снимает выделение с остальных кнопок.  
сеттер `address(value: string)` — устанавливает значение в текстовое поле адреса.

#### Класс ContactsForm

Управляет второй формой оформления заказа (ввод электронной почты и номера телефона).
_Наследуется от базового класса `Form`._

#### Класс Success

Управляет интерфейсом финального окна, уведомляющего об успешном совершении покупки и списании средств.

## Слой презентера (Presenter)

Взаимодействие между Моделями данных и компонентами Представления (View) реализовано в главном скрипте `src/main.ts` при помощи брокера событий `EventEmitter`. Презентер подписывается на события интерфейса и моделей и координирует их работу: вызывает методы моделей и обновляет представления, но сам не хранит состояние и не генерирует события об изменении данных — эту ответственность несут модели.

### Список генерируемых и обрабатываемых событий:

- `items:changed` — генерируется моделью продуктов (`ProductsData`) при успешном обновлении или инициализации списка товаров. Инициирует перерендер витрины интернет-магазина.
- `card:select` — генерируется компонентом карточки товара при клике пользователя. Презентер по этому событию записывает выбранный продукт в модель продуктов (`productsData.preview`).
- `preview:changed` — генерируется моделью продуктов (`ProductsData`) при записи товара в поле превью. Открывает модальное окно с подробным превью товара.
- `card:preview-toggle` — генерируется при клике на кнопку добавления/удаления товара внутри превью-карточки. Презентер по этому событию вызывает `add`/`remove` модели корзины; сама корзина при этом сообщает об изменении своего состава событием `basket:changed`.
- `basket:open` — вызывается при клике на иконку корзины в шапке профиля. Открывает модальное окно корзины с уже актуальным содержимым, без дополнительного обновления данных.
- `basket:changed` — генерируется моделью корзины (`BasketData`) при любом изменении её состава (добавление, удаление, очистка). Синхронизирует счётчик товаров в шапке и список элементов внутри модального окна.
- `basket:delete-item` — генерируется при клике на кнопку «удалить» (иконка корзины/мусорки) у конкретного товара внутри списка покупок. Презентер вызывает `remove` модели корзины, которая сама оповещает об изменении.
- `order:open` — открывает первый шаг формы оформления заказа (выбор способа оплаты и ввод адреса доставки).
- `order.payment:change` / `order.address:change` — сигнализируют о вводе или переключении данных на первом шаге оформления заказа. Презентер передаёт значения в модель покупателя (`UserData.setField`).
- `order:submit` — срабатывает при отправке первой формы, переключая модальное окно на второй шаг.
- `contacts.email:change` / `contacts.phone:change` — сигнализируют об изменении контактных данных пользователя. Презентер передаёт значения в модель покупателя (`UserData.setField`).
- `buyer:changed` — генерируется моделью покупателя (`UserData`) при изменении любого из её полей или при очистке данных. Презентер по этому событию производит валидацию полей и динамически управляет активностью кнопок отправки (`Далее` / `Оплатить`) и выводом ошибок.
- `contacts:submit` — отправляет сформированный заказ на сервер, очищает модели корзины и покупателя и открывает окно успешного завершения операции. Очистка корзины сама инициирует обновление счётчика и списка через `basket:changed`.
- `success:close` — закрывает окно успешного оформления заказа.
- `modal:open` / `modal:close` — управляют блокировкой скролла основной страницы при работе с модальными окнами.
